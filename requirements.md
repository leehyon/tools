# 工具收集平台 - 设计需求文档

## 项目概述

开发一款轻量级、现代化的工具收集展示平台（域名地址 tools.kohsruhe.com），用于记录和展示各类实用 App / 工具信息，支持标签分类、搜索筛选、数据统计等核心功能，采用静态 JSON 存储数据。

## 核心功能需求

数据来源：基于 public/data.json 结构化存储的工具信息

### 前端核心功能

- 数据展示：以卡片式布局展示所有工具信息，包含名称、简介、标签、访问链接等
- 搜索筛选：支持按工具名称、简介、标签关键词实时过滤
- 数据统计：展示核心数据指标（总工具数、唯一标签数）

## UI/UX 设计要求

参考 附图 管理后台风格

配色：简洁
布局：左侧固定侧边栏 + 右侧主内容区，顶部通栏包含搜索和操作按钮；
组件风格：卡片圆角 12px，轻微阴影，无冗余装饰

### 页面结构

侧边栏：导航菜单包括总工具数量，分LIST的工具类别

顶部通栏：搜索框

数据区：
- 每月工具趋势
- 各个类别的树状图
- 工具表单，类似Notion的表
  
## 技术实现

技术栈：
前端：Vite + 框架

性能与兼容性
响应式适配：支持移动端（768px 以下）、平板、桌面端；
加载性能：静态资源按需加载，无冗余依赖；
浏览器兼容：支持现代浏览器（Chrome、Firefox、Safari 最新版）。

工程化要求
构建输出：Vite 打包至 dist 目录；
触发条件：仅当 data.json 文件变更时触发部署；

## 交付物要求
完整可运行的前端代码（包含 HTML/CSS/JS/JSON）；

本地运行说明（npm run dev）、部署说明

代码规范：无冗余代码，关键逻辑添加注释，变量 / 函数命名为英文语义化名称
